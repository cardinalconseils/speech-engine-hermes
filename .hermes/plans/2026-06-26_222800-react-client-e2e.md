# Implementation Plan: Speech Engine React Client + E2E Tests

**Date:** 2026-06-26
**Rubric:** `.hermes/rubrics/2026-06-26-react-client-e2e.md`
**Status:** Planning

## Architecture

```
client/
├── index.html              # Vite entry HTML
├── package.json            # Client dependencies
├── tsconfig.json           # TypeScript config
├── vite.config.ts          # Vite config with proxy
├── src/
│   ├── main.tsx            # React entry point
│   ├── App.tsx             # Root component
│   ├── App.css             # Styles
│   ├── components/
│   │   ├── MicButton.tsx   # Animated mic button with state management
│   │   ├── StatusDisplay.tsx # Connection status text
│   │   └── ErrorDisplay.tsx  # Error message with retry
│   ├── hooks/
│   │   └── useConversation.ts # ElevenLabs conversation hook wrapper
│   └── lib/
│       ├── tokens.ts       # Token fetching from token server
│       └── mediaCheck.ts   # Browser capability detection
└── e2e/
    ├── playwright.config.ts # Playwright config
    ├── fixtures.ts         # Test fixtures
    └── speech-engine.spec.ts # E2E test spec
```

## Tasks

### Task 1: Scaffold Vite + React + TypeScript project
- Create `client/package.json` with dependencies (react, react-dom, @elevenlabs/react, vite, typescript)
- Create `client/tsconfig.json`
- Create `client/vite.config.ts` with proxy to token server (port 3002)
- Create `client/index.html` as Vite entry point
- Create `client/src/main.tsx` as React entry point
- Run `npm install` in client/

### Task 2: Build core React components
- `App.tsx` — root component with layout
- `App.css` — dark theme styling matching existing design
- `MicButton.tsx` — animated mic button with state:
  - idle (grey circle, mic icon)
  - connecting (pulsing animation, spinner)
  - connected (red circle, pulsing)
  - error (show error state)
- `StatusDisplay.tsx` — status text below button
- `ErrorDisplay.tsx` — error message with retry button

### Task 3: Browser capability detection
- `mediaCheck.ts` — check for navigator.mediaDevices, getUserMedia, etc.
- Show graceful error if WebRTC/audio unavailable

### Task 4: ElevenLabs conversation integration
- `tokens.ts` — fetch token from token server with error handling
- `useConversation.ts` — hook wrapping @elevenlabs/react useConversation
- Wire up in App.tsx with state management

### Task 5: Wire everything together
- Connect MicButton clicks to conversation start/end
- Handle loading states during token fetch
- Handle error states with clear messages
- Prevent concurrent connections
- Clean up conversation on unmount
- Build verification: `npm run build` succeeds

### Task 6: Write Playwright e2e tests
- Install Playwright: `npm install -D @playwright/test`
- `playwright.config.ts` — config with env-variable base URLs
- `speech-engine.spec.ts` — test covering:
  - Page loads and renders UI (Criterion 1)
  - Error handling when token server down (Criterion 2)
  - Button state prevents double-click (Criterion 3)
  - Configurable base URLs (Criterion 4)
  - Graceful degradation mock test (Criterion 5)

### Task 7: Auto-QA verification
- Start dev servers (WebSocket on 3001, token on 3002, client on 5173)
- Run browser-based verification against rubric
- Run Playwright e2e tests
